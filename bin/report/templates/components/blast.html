Homology to viral reference sequences

This section shows the results from a BLAST homology search that was performed on a contig set generated using the de novo assembly tool SPADES.
The table below shows contigs that were at least 150 bp long and matched a known virus/viroid sequence, when performing a homology search against the core nucleotide database.
It can be sorted by alphabetical or numerical order by clicking on the column headers, and the number of entries shown can be adjusted using the dropdown menu above the table.
The results can be limited to a specific virus genus or species by typing its name in the search box.
The table can also be filtered to only show rows which meet different criteria (i.e. minimum coverage, terminology, and/or best contig per species).
A representative contig (the "best contig per species") is selected based on the preliminary blast homology search. Contigs assigned to the same species are ranked using a weighted indicator-based composite score that rewards several key features (% identity, bitscore, evalue, query coverage per subject, k-mer coverage of contig during assembly, contig length, alignment length, and annotation completeness). The contig with the highest score is then selected as the representative for that species.
The reads are also mapped back to the contigs to derive statistics on the number of reads mapped, mean depth, % coverage at 30X, and mean mapping quality. A confidence score is calculated based on these statistics to evaluate how well the raw sequencing data supports the assembled contigs.
